# 울릉군 생태관광 AI 교육 — 커리큘럼 순서 전면 재편 설계

- 날짜: 2026-07-03
- 대상 파일: `ulleung-ecotourism-edu/index.html` (단일 파일 HTML)
- 배포: `island/main` push → GitHub Pages (`kanghoon1204.github.io/Ulleungdo_Island/`)

## 1. 목적

현재 19개 섹션으로 흩어진 수업 흐름을, 개념 → 예시 → 실습 → 배포로 이어지는
**10단계 흐름**으로 재편한다. 핵심 원칙은 **"읽기 전에 하기(do-before-read)"** —
읽을 분량을 줄이고 참가자가 빨리 손을 움직이게 하며, 실습 결과물이 버려지지 않고
그날 만드는 웹페이지의 부품이 되도록 묶는다.

대상: 울릉군 생태관광 전문인재(홍보 담당·현장 가이드·행정 등 배경이 제각각).

## 2. 새 10단계 구조

| # | 섹션 | 핵심 내용 | 현재 섹션에서 이동 |
|---|------|-----------|-------------------|
| 1 | **AI 이해** | AI는 초안 도우미 + 기대치(다 해주진 않는다) — 짧게 | `whatai` |
| 2 | **예시** | NotebookLM(데모) · Suno·포스터(미니체험) · 웹페이지(오늘 메인) | `goal`+`taste` 재편 |
| 3 | **시작 준비** | 도구 소개 + Gemini·ChatGPT 켜기 | `tools`+`setup` |
| 4 | **프롬프트 엔지니어링** | "같은 질문도 답이 다르다" + 좋은 질문 기본 — 얇게 | `step1` |
| 5 | **실습** | A 웹 재료(필수) → B 다양한 실습 메뉴(선택) | `workspace`+`step2`+`step3`+`step5`+`can` |
| 6 | **웹페이지 만들기** | index.html 코드 받기 + 완성 체크리스트 | `step7`+`checklist` |
| 7 | **깃허브 배포하기** | GitHub Pages 배포 | `publish` 전반부 |
| 8 | **QR코드 변환하기** | QR 생성 | `publish` 후반부 |
| 9 | **부록** | 프롬프트 템플릿 + 체이닝 + AI 흐름 심화 + 마무리 | `templates`+`step6`+`deepai`+`end` |
| — | **진행 바(상단)** | "N/9단계" breadcrumb — 삭제된 roadmap 대체 | `roadmap` 삭제 |

### 제거

- **작업 흐름(`roadmap`) 섹션 삭제** — 섹션 진행 자체가 순서를 보여주므로 중복.
  대신 상단에 얇은 진행 바/breadcrumd로 "현재 위치"만 남긴다.

### 배치 판단 근거

- **`can`(현장 활용 사례) → 5번 실습 B로**: `can`은 "활용 사례 리스트"인데,
  그 사례를 실제로 해보는 곳이 실습 B다. 리스트와 실습이 분리되지 않도록 합친다.
  1번 AI 이해는 개념·기대치 한 줄만 남겨 가볍게 유지.
- **`step6`(체이닝) → 9번 부록으로**: 체이닝은 초보자에겐 심화. 이미 부록으로 뺀
  `templates`·`deepai`와 난이도가 같은 급. 4번 프롬프트 섹션은 얇게 유지.

## 3. 무게중심 섹션 상세

### 2번 · 예시 (4카드)

각 카드 구조: **뭐냐 → 현장 쓸모 → (체험이면) 3분 프롬프트**

| 예시 | 성격 | 구성 |
|------|------|------|
| NotebookLM | 데모 | 자료 → 대화형 음성 요약. 강사가 미리 만든 "울릉도 음성 가이드" 재생만 |
| Suno | 미니체험 | 예: *"울릉도 오징어 홍보 신나는 트로트, 후렴에 '울릉도' 반복"* → 재생 |
| 포스터 이미지 | 미니체험 | 예: *"울릉도 일출과 독도, 여행 홍보 포스터, 따뜻한 색감, 세로형"* → 저장 |
| 울릉도 홍보 웹페이지 | 오늘 메인 | 완성 예시(`example.html`) 미리보기 → "오늘 이걸 만들어 QR까지" |

의도: Suno·포스터로 초반에 "3분 만에 내 결과물 하나"를 손에 쥐어 작은 성공을
만들고, 뒤의 준비·프롬프트 구간을 버티는 동력으로 삼는다.

### 5번 · 실습 (A 필수 + B 선택 메뉴)

**A. 웹페이지 재료 만들기 (순서대로, 결과물이 6번으로 넘어감)**

| 단계 | 만드는 것 | 현재 섹션 |
|------|-----------|-----------|
| A1 | 문장 모으기 워크스페이스 — AI 답변 중 쓸 문장만 골라 모음 | `workspace` |
| A2 | 소개 글 재료 — 홍보 문구, 코스 설명, 한 줄 슬로건 | `step2` |
| A3 | 다국어 안내 — 영·중·일 방문 안내문 | `step3` |
| A4 | 방문자 질문·확인 항목 — FAQ, 준비물 체크 | `step5` |

**B. 다양한 실습 메뉴 (골라서 1~2개 체험)** — `can` 흡수

각자 업무에 맞는 걸 골라 직접 해보는 메뉴. 전부 하지 않는다.

| 실습 | 만드는 것 |
|------|-----------|
| SNS 홍보 글 | 인스타·블로그 게시글(해시태그) |
| 관광 코스·상품 소개 | 추천 코스 글, 한 줄 카피 |
| 다국어 안내 | 영·중·일 홍보·안내 문구 |
| 투어 멘트 | 외국인 맞이 인사, 코스 설명 스크립트 |
| 문의·민원 답변 | 정중한 답변 초안 |
| 보도자료·이메일 | 행사 알림, 제휴처 발송 글 |

의도: 참가자 배경이 제각각이어도 각자 쓸모 있는 게 하나는 있게. 강사는 하나만
시연, 나머지는 템플릿으로 열어둔다. A는 다 같이(필수), B는 골라서(선택)로 나눠
진도 관리도 쉽게.

## 4. 구현 시 유의

- 단일 파일 HTML 원칙 유지(`<style>`/`<script>` 인라인), 기존 디자인 토큰 재사용.
- 섹션 `id`와 상단 내비게이션 링크(`href="#..."`)를 새 순서·명칭에 맞춰 전면 갱신.
- 기존 콘텐츠는 최대한 재사용하고 이동·재배치 위주. 신규 작성은 예시 4카드 정비와
  실습 B 메뉴가 중심.
- `index_backup_실습판.html`은 백업으로 보존.

## 5. 성공 기준

- 상단 내비게이션이 새 10단계(진행 바 포함)를 정확히 반영하고 앵커 이동이 동작.
- 실습 A 결과물이 6번 웹페이지 만들기로 자연스럽게 이어지는 서술.
- 모바일 반응형 유지, 깨진 링크·중복 섹션 없음.
- 삭제 대상(`roadmap`) 제거, 이동 대상(`can`→실습B, `step6`→부록) 반영.
